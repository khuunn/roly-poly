현재 세션에서 변경된 모든 파일에 대해 다음 7가지 관점으로 종합 리뷰를 수행하세요.
git diff를 통해 변경사항을 확인하고, 각 항목별로 판정 결과를 보고하세요.

## 리뷰 체크리스트

### 1. 목적 부합
- 요구사항 대비 구현이 정확한지
- 빠진 기능이나 과잉 구현은 없는지

### 2. 보안 / 버그
- OWASP Top 10 취약점 (injection, XSS 등)
- 잠재적 런타임 에러 (null reference, 경계값 등)
- 크리티컬한 로직 오류

### 3. 사이드 이펙트
- 기존 기능에 의도치 않은 영향은 없는지
- 공유 상태나 전역 변수 변경 여부
- 기존 테스트가 깨질 가능성

### 4. 코드 구조
- 단일 함수 50줄 초과 시 분리 검토
- 단일 파일 300줄 초과 시 분리 검토
- 책임이 명확히 분리되어 있는지

### 5. 재사용 / 통합
- 기존 유틸리티나 헬퍼와 중복되는 코드는 없는지
- 공통 로직을 추출할 수 있는 부분은 없는지

### 6. 정리
- 구현 과정에서 불필요해진 import, 변수, 함수 제거
- 디버깅용 코드 (print, console.log 등) 제거
- 주석 처리된 코드 제거

### 7. 코드 품질
- 네이밍 컨벤션 일관성
- 에러 핸들링 적절성
- 타입 안전성 (해당되는 경우)

## 출력 형식

각 항목별로 다음 형식으로 보고하세요:

| # | 항목 | 판정 | 비고 |
|---|------|------|------|
| 1 | 목적 부합 | OK / ISSUE | 구체적 내용 |
| ... | ... | ... | ... |

ISSUE가 있는 항목은 구체적으로 문제와 수정 방안을 설명하세요.
사용자가 동의하면 직접 수정을 진행하세요.
